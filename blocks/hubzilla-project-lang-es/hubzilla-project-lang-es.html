Red descentralizada de hubs: la red

Los hubs son los servidores que se comunican entre sí para propagar información a través de la red, o "grid". No hay centro central o punto único de falla. Los hubs se pueden conectar a tantos o tan pocos hubs como se desee.

Canales y Conexiones

Un canal es la identidad fundamental en la red. Un canal puede representar muchas cosas diferentes: una persona, un blog o un foro para nombrar algunos. Los canales pueden establecer conexiones con otros canales para compartir información con permisos altamente detallados. Los canales se dirigen usando un formato familiar channel@hub.domain.

Identidad nómada y clonación

La identidad nómada significa verdadera propiedad de la identidad en línea. Con Hubzilla, no tienes una cuenta en un servidor, tienes una identidad que puedes llevar contigo a través de la cuadrícula. Puede clonar un canal a través de varios concentradores para resiliencia contra fallas de red o censura, o puede mover completamente un canal de un concentrador a otro, llevando sus datos y conexiones con usted.

Control de acceso a través de la web

Los sitios web típicos están aislados y no tienen idea de quién está accediendo a su contenido y el acceso controlado a los datos se limita a la configuración de permisos entre cuentas individuales de un sitio. Si quieres compartir información de forma controlada fuera del sitio, no tienes suerte. Hubzilla impone permisos finos para la información compartida a través de la red, y los sitios web en los concentradores son conscientes de la identidad. El inicio de sesión único permite una autenticación sin fisuras en sitios web independientes de una manera nunca antes posible.

¿Tienes Zot?

Zot es un framework web basado en JSON para implementar comunicaciones y servicios descentralizados seguros. Se diferencia de muchos otros protocolos de comunicación mediante la construcción de comunicaciones sobre un marco de identidad y autenticación descentralizada. El componente de autenticación es similar a OpenID conceptualmente, pero está aislado de las identidades basadas en DNS. Cuando es posible, la autenticación remota es silenciosa e invisible. Esto proporciona un mecanismo para el control de acceso distribuido a escala de Internet que es discreto.

  • Comunicaciones completamente descentralizadas
  • Aislamiento de identidad basada en DNS
  • Movilidad del nodo
  • Autenticación remota invisible o reducida de "interacción"
  • rendimiento

Plugins

Hubzilla es extensible. Construir aplicaciones web completas como complementos y aprovechar la potencia de la red para manejar el control de acceso y la distribución de datos.

Kit de herramientas para la creación de sitios web

Utilice el lenguaje de descripción de páginas web de Comanche y la herramienta Páginas Web para crear elementos de página web modulares, que tengan en cuenta la identidad y sean compartibles.

Redes sociales

Las redes sociales están en el ADN de Hubzilla, cocido en su núcleo. Utilice un canal para representarse y hacer conexiones (es decir, "amigos") para compartir con otros. Soporte para federaciones con Diaspora, Friendica y otras personas a través de plugins.

Almacenamiento de archivos en nube

Conecte el almacenamiento de archivos de su canal a su sistema de archivos local utilizando WebDAV estándar. Compartir o publicar sus archivos con el control de acceso de grano fino omnipresente.

Temas

La interfaz web es altamente personalizable, con soporte nativo para el desarrollo de temas. Marca tus hubs para tu comunidad o negocio.

Eventos

Coordine y comparta eventos de calendario con cualquiera que utilice la funcionalidad de calendario nativo. Importación / exportación entre formatos de calendario populares.

API

Una lista creciente de funciones API permite a las aplicaciones independientes utilizar concentradores, ampliando la gama de servicios interconectados en la red hasta los límites de su imaginación.

Libertad

Hubzilla es de código abierto! Usted tiene la libertad de usar y modificar el software para satisfacer sus necesidades. Cuando se trata de privacidad y seguridad de datos, todo el mundo debería ser capaz de revisar y mejorar el código.
