Децентрализованная сеть из хабов: грид

Хабы это серверы, которые обмениваются данными в сети "grid". Все хабы равноправны. Хабы могут быть в зависимости от потребности связаны с одним или более сторонними хабами.

Каналы и контакты

Канал - это основной элемент в гриде. Канал может представлять множество различных событий, например человека, блог или форум. Каналы могут связываться с другими каналами для обмена информации в соответствии с прецизионными правами доступа. Адрес канала имеет привычный формат channel@hub.domain.

Прозрачная миграция и клонирование

Прозрачная миграция означает настоящий контроль над своими данными. В Хабзилле ты заводишь аккаунт в гриде, а не на определённом сервере и этот аккаунт может перемещаться по всему гриду. Ты можешь завести два или больше аккаунта, чтобы улучшить устойчивость против цензуры и доступность, ты можешь переместиться с одного хаба на другой вместе с данными и с контактами.

Контроль доступа во всей сети

Типичный сайт изолирован и не имеет понятия, кто посещает его и контроль доступа ограничивается контролем для зарегистрированных на определённом сайте пользователей. Если вы хотите предоставить доступ к информации только определённым пользователям, не зарегистрированным на сайте, то вам не повезло. Хабзилла же предоставляет прецизионный доступ к информации, распространяемой в сети, и сайты на хабах могут распознавать посетителей. Благодаря Single sign-on возможна простая аутентификация на независимых сайтах, что раньше было невозможно.

Что такое Zot?

Zot - это базирующийся на JSON набор функций для реализации децентрализованной коммуникации и децентрализованных сервисов. Отличительные свойства Zot в сравнении с многими другими коммуникационными протоколами - это прозрачная миграция и набор функций для аутентификации. Аутентификационная составляющая Zot подобна OpenID, но изолирована от идентификаторов на основе DNS. Причём возможна тихая и невидимая удалённая аутентификация. Таким образом реализуется механизм для распределённого доступа через интернет.

  • полностью децентрализованная связь
  • изоляция от DNS зависимых идентификаторов
  • узел мобильности
  • невидимый или компактный обмен при удалённой аутентификации
  • быстродействие

Дополнение

Hubzilla постоянно развивается. Программируй веб-приложения как дополнения и дополни возможности грида для передачи данных и управления доступом.

Набор инструментов

Язык Comanche используется для описания интернет-страниц и для программирования модульных, реагирующих на конкретного пользователя элементов страниц. Эти модули могут быть использованы другими hubzilla-пользователями.

Социальная сеть

Свойство социальности записано в ДНК Хабзиллы, в её ядре. Используйте каналы для того, чтобы рассказать о себе и связаться с другими участниками Хабзилла грида. Возможен обмен сообщениями с социальными сетями Диаспора, Фрэндика и др. при помощи плагинов.

Облачное хранилище файлов

Соедините облачное хранилище файлов вашего канала с вашим жестким диском, используя технологию WebDAV. Распространяйте или публикуйте ваши файлы при помощи вездесущной прецизионной системой управления доступом.

Внешний вид

Внешний вид может быть гибко изменён с помощью встроенной поддержкой тем. Сделайте ваш Хаб неповторимым и подходящим для вашего сообщества или вашей фирмы.

События

Координируйте и распространяйте календарные события cо встроенной поддержкой функции календаря. Импорт и экспорт с поддержкой распространённых форматов.

API

Богатое и гибкое API позволяет создавать независимые приложения для использования Хабов, расширения выбора сетевых сервисов в гриде на основе вашей фантазии.
